1. Project Overview

The ClimateHealth Intelligence Platform (Climate Disease Predictor) is a full-stack AI/ML-powered web application that predicts potential disease outbreaks based on real-time climate and environmental data. It combines machine learning models, rule-based expert systems, and external weather/health APIs to provide accurate, explainable predictions with actionable health precautions.

üéØ Core Goal: Bridge the gap between climate science and public health by enabling users to input environmental parameters (Temperature, Humidity, Rainfall, AQI) and receive AI-powered disease risk assessments with full transparency into how the prediction was made.

The system predicts 8 disease categories:

Training Process (train_model.py):

  • Dataset: 5,000 synthetically generated samples using domain-expert probability distributions
  • Features: 4 inputs ‚Äî Temperature (¬∞C), Humidity (%), Rainfall (mm), AQI (index)
  • Target: 8 disease classes
  • Split: 80% train / 20% test with stratified sampling
  • Validation: 5-fold cross-validation for each model
  • Best model is automatically selected and saved as model.pkl

5. Prediction Pipeline ‚Äî Hybrid AI Engine

The prediction system is NOT a simple classifier. It uses a sophisticated 3-stage hybrid pipeline:

Stage 1: Ensemble Model Voting

All 3 ML models (RF, SVM, LR) are loaded. Each produces probability distributions across 8 diseases. These probabilities are accumulated and averaged.

Stage 2: Expert Rule Engine

A comprehensive rule-based system encodes medical domain knowledge:

  • Malaria rules: Temp >25¬∞C + Humidity >60% + Rainfall >100mm
  • Dengue rules: 25‚Äì35¬∞C + Humidity >60% + Standing water conditions
  • Asthma rules: AQI >100, with escalating thresholds at 150, 200, 300
  • Heat Stroke rules: Temp >38¬∞C, especially with low humidity
  • Regional Risk Boost: High-risk locations get boosted vector-disease scores

Stage 3: Fusion & Decision

Final Score = (ML Probability √ó 0.6) + (Rule Score √ó 0.4)
This weighted fusion ensures the system benefits from data-driven ML while respecting established medical knowledge.

Stage 4: Explainable AI Engine

After prediction, the system generates transparency data:

  • RF Feature Importances: Extracted from model.feature_importances_
  • Rule Contributions: Measures how current values triggered expert rules
  • Combined Importance: 50/50 fusion of ML + rules, normalized to 100%
  • Natural Language Reasoning: Human-readable sentences explaining each factor
  • SHAP-style Waterfall: Step-by-step risk buildup from 12.5% base to final score
